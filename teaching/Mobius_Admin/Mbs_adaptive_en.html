How to create Adaptive Assignments

Introduction

We give a summary of the steps to create an adaptive assignment. Most of the steps are identical to those used to create a standard assignment. We will therefore regularly refer to the documents and .

An adaptive assignment is composed of several branches. Each branch is of a different level of difficulty. You select the branch that students start with. Students move to a branch of lower of upper level of difficulty according to the policies that you set.

Procedure

  1. As for creating an assignment, you have to access the folder under Content repository ==> Assignments where you would like to store the adaptive assignment. Unless you have an elaborate hierarchy of assignments, you will probably store your assignments in the folder Default Unit (item 1 in the figure bellow).

    There is presently no assignment in the folder Default Unit depicted above.

  2. To begin editing your new adaptive assignment, you have to select the item Adaptive Assignment (item 2 in the figure below) that you can find under the pull down menu Create New at the bottom of the folder that you have selected.

  3. The first tab to create an adaptive assignment is called 1. Choose name (item 3 in the figure below) and it is identical to the first tab to create a standard assignment. A complete description of this tab is given in item 6 of the procedure to create a standard assignment in .

  4. The real work to create an adaptive assignment is done under the tab 2. Select Questions (item 4 in the figure below). Initially, there is only one branch which is the starting branch (item 5 in the figure below). Using the drop down menu on the right side of the title of the branch, we may import questions, create new question and add new question groups to this branch. We will come back to this shortly. For now, we want to mention that you can select the item Properties (item 6 in the figure below) to modify the properties of the branch including its title.

  5. If you select the item properties, you get the following popup window.

    • You may give a name to each branch that you create.
    • You may select the branch that will be the starting branch. To change the starting branch for the assignment, you have to check the box beside Start Branch for the branch that you want to be the starting branch. It is not possible to deselect the option Start Branch. Think of these boxes beside the Start Branch options for each branch as radio boxes (for those old enough to remember radios in old cars). There must always be one and only one starting branch.
    • You can assign the weight that each question of this branch will have for the computation of the finale grade. Questions in an easy branch may be worth less than questions in a more challenging branch.
    • You can also select to scramble the questions. By default the question are assigned to the students in the order that they are listed in the branch. If you select this option, the questions are scrambled before being assigned to the student. If all the questions of the branch are used before the student exits the branch, the questions will be scrambled again before being assigned to the student. If you don't request that questions be scrambled, the questions will again be assigned to the student in the exact order that they are listed in the branch. This is repeated as long as the student does not exit the branch.
    • You can request that question be recycled. If a student needs to request more questions than there are in a branch, this option is automatically activated (if it was not) with the restriction that all questions of the branch must be used before restarting to use them. If this option is selected, the next question assigned to a student is chosen from all the question with a chance that the same question be repeated.

    Do not forget to save you modifications when you are done (item 7 in the figure above).

  6. To add content to a branch, you may use the drop down menu at the right of the title of a branch as we mentioned above or, if the branch is selected, you can use the main menu on the right of the second tab as illustrated below.

    A complete description of the procedure to select questions and question groups is given under item 7 of the procedure to create a standard assignment in . We will not repeat this procedure here.

  7. To add branches to your assignment, click on the button Add Branch at the bottom of the Navigation area that you can find on the left margin of the second tab (item 8 in the figure below) and proceed as we have done above to build your new branch.

  8. Let's assume that we have all the branches for our assignment and that you have selected your starting branch. It is now time to define the policies for transferring from one branch to the other. To define the policies, you have to click on the item Policies that you can find above the Navigation area on the right margin of the second tab (item 9 in the figure above).

  9. After clicking on the item Policies, you get the following menu.

    There are four sections to this menu. The first section, Grade Assignment When, gives you three options to choose from.

    1. Number of Questions Seen (i.e. number of questions requested by the student)
    2. Number of Correct Responses
    3. Number of Incorrect Responses

    The following rules are applied:

    • You can select one or more options.
    • The first option to be satisfied ends the assignment. The assignment is then graded.
    • The number of questions in the assignment is the the largest number chosen for the selected options. For instance, if the assignment is to end when 10 questions have been seen or 7 correct answers have been given, then the assignment will have 10 questions.
    • A question is counted as a Correct Response if the student get all parts of the question right. A wrong answer to one part of the question implies that the full question is counted as an Incorrect response.

    The second section, Grade Policy, offers three options.

    1. Basic
    2. ELO
    3. Algorithmic

    For the Basic option, the assignment is graded based on the weight that you have assigned to each question of each branch. The ELO policy is a method for calculating the relative skill levels of players in zero-sum games (consult the sites and formor einformation). If you select the option Algorithmic, then the option Algorithmic in the following section of the menu is automatically selected. You then have to create an algorithm that end with the command returnGrade(N), where N is the grade computed according to your algorithm. We will address the option Algorithmic in a section below.

    The third section of the menu is Adaptive Policy and offers two options.

    1. Basic
    2. Algorithmic

    The Basic option (default) gives you the menu in the fourth and last section of the menu. This is the one that we will describe right now. The option Algorithmic will be the subject of the section below.

    You have the option to transfer a student to the next higher level of difficulty if the student answers correctly a certain number of questions (consecutively or non-consecutively according to your choice), or to transfer a student to the level of difficulty below if the student fails to answer correctly a certain number of questions (again consecutively or non-consecutively according to your choice).

    If there are not enough questions in a branch to meet the requirements to transfer to another branch, questions are repeated.

  10. Use the button Back on the top left corner of the tab (item 10 in the previous figure) to go back to the full tab 2. Select Questions. There is no save required at this time.

  11. To complete the adaptive assignment, you have to add the properties. Will the assignment be graded? What will be the deadline to submit the assignment? Etc. To do so, select the third tab, the tab entitled Properties (item 11 in the figure below).

  12. The tab Properties for the adaptive assignments is almost identical to the tab Properties that is described on the site for the standard assignments. As for the tab Policies for the standard assignments, the tab Policies for the adaptive assignments contains the mains sections: Types of Assignments, Policy Set, Scheduling and Visibility, Feedback (After), Reporting and IP Address/Host Names Whitelist. The main differences are that the main section General Properties in the standard assignment is not present for the adaptive assignments, and there is one additional main section for the adaptive assignments called Feedback (During) (item 12 in the figure below).

    There is a little difference between the main section Types of Assignments for the adaptive and standard assignments. For the adaptive assignment, there is no option Mastery Dialog and Study Session Dialog. All the other common main sections of the adaptive and standard assignments are identical. We refer the reader to the document for the detailed instructions.

    The new main section Feedback (During) (item 12 in the figure above) is really short. You can activate this option to provide feedback to the students during their assignment. They get a little table at the top of their assignment that shows their progress: on which branch of the assignment they are, the question that they got right and those they got wrong so far, and the number of questions answered.

    After submitting their assignment, students get a detailed summary of their results that includes the list of questions answered correctly and incorrectly for each branch, their rough score for each branch and their final score computed according to the algorithm selected by the instructor. In the figure below, the algorithm used is the one presented in the below.

    13. Algorithmic Policies

    You may create a Maple algorithm to choose the branch from which the next question should be drawn from and, possibly, to determine when an assignment will end.

    The algorithm must return one of the following three values:

    • The number of the next branch to draw the question from. The end of the assignment is determined by the conditions that you have selected in the first section of the menu for the Policies.
    • The variable GRADE. This will end the assignment and compute the final grade according to the weight that you have associated to the questions of each branch.
    • A call to the procedure returnGrade(N), where N is the final grade for the assignment computed according to your algorithm. As stated previously, this procedure must be used when the option Algorithmic is selected for grading the assignment in the second section of the menu for the Policies.

    To create an algorithm, you may use one of the following variable:

    Variable Description
    num_branches The number of branches.
    branch_names A list with num_branches elements. The first element is the name of the first branch, the second element is the name of the second branch, and so on.
    branch_weights A list with num_branches elements. The first element is the weight of each question in the first branch, the second element is the weight of each question in the second branch, and so on.
    start_branch The number of the starting branch.
    current_branch The number of the current branch; namely, the number of the branch used to draw the last question requested by the student.
    seen The number of questions a student has requested (seen) so far.
    responses A list with seen elements. The first element is 1 if the student has gotten the right answer (all parts of the question have been answered correctly) for the first question and 0 otherwise (a wrong answer to a single part of the question results in 0), the second element is 1 if the student has gotten the right answer for the second question and 0 otherwise, and so on.
    response_branches A list with seen elements. The first element is the number of the branch from which the first question was drawn, the second element is the number of the branch from which the second question was drawn, and so on.
    grades A list with seen elements. The first element is the student's grade (from 0 to 1) for the first question, the second element is the student's grade (from 0 to 1) for the second question, and so on. This is different than response_branches because partial grades are possible.
    correct The number of right answers from all the branches.
    incorrect The number of wrong answer from all the branches.
    streak The number of correct/incorrect consecutive answers counted from the last question requested by the student. If the number is positive, it is the number of consecutive correct answers. if the number is negative, it is the number consecutive incorrect answers.
    GRADE The value to return when it is time to grade the assignment using the basic grading option.

    If an error occurs, the system will use the basic policy to choose the next branch.

    Example 1

    This example is from DigitalEd. In this example, the assignment has 12 questions and three branches: easy, medium and difficult. The final score for the assignment is one less than the number of the branch where the next question would have come from. The assignment may end before 12 questions are drawn if the student answers incorrectly two consecutive questions drawn from the branch associated to the lowest level of difficulty. The student then get 0 for the assignment (one less than the branch 1 that should have been the next branch used.) The assignment also ends before 12 questions are drawn if the student answers correctly two consecutive questions drawn from the branch associated to the highest level of difficulty. The student then get the maximal score num_branches for the assignment (one less than the fictional branch num_branches + 1 that should have been the next branch used.)

    After two consecutive right answers, students are bumped up to the branch just above the current branch. After two consecutive wrong answers, students are dropped to the branch just below the current branch.

    It is not given by the algorithm but we may assume that students start at the lowest level and their goal is to end as the highest level as possible.

    
nextBranch := current_branch:
if abs(streak) >= 2 and response_branches[seen-2+1] = current_branch then
    nextBranch := current_branch + signum(streak):
end if:
score := max(0, nextBranch - 1):
if nextBranch >= 1 and nextBranch <= num_branches and seen<12 then
    nextBranch, score;
else
    returnGrade(score);
end if;

    For the first test, we verify first if we have two consecutive right answers or two consecutive wrong answer. If we have a consecutive sequence of two right (wrong) answers, then we check that these two consecutive right (wrong) answers come from the same branch. For that, we only have to check that the first question in the consecutive sequence considered comes from the same branch than the current question. The student could not have moved to another branch and come back to the initial branch during the consecutive sequence presently considered because that would have required at least two consecutive sequences, one of right answers and one of wrong answers, in total longer than the consecutive sequence presently considered. If all the conditions are satisfied, the next question will be drawn from the branch which is one up if streak is positive or one lest is streak is negative.

    For the second test, we verify if the student can request another question. If the students does not have two consecutive wrong answers at the lowest level of difficulty (i.e. nextBranch < 1) or two consecutive right answers at the highest level of difficulty (i.e. nextBranch > num_branches), and has requested less than 12 questions, then the student can request another question. Otherwise, the assignment ends.

    In the code above, the current score is also returned with the number of the next branch (see the line nextBranch, score;). This score is used when the option Show Current Grade of the Policy Set of this assignment has been selected.

    Example 2

    The following example was inspired by an example given in the document .

    This assignment has four branches. Students should answer correctly 3 consecutive questions from the first branch and 3 consecutive questions from the third branch to get the maximal score. Students can request no more than 20 questions to do that. Students start in the first branch and are transferred to the third branch only after having answered correctly 3 consecutive questions from the first branch.

    Students are transferred to the second branch if they fail to answer correctly 2 consecutive questions in the first branch. It is assumed that questions in the second branch are easier than those in the first branch and help students prepare for the questions in the first branch. Students are transferred back to the first branch after having answering correctly 3 consecutive questions from the second branch.

    Similarly, students are transferred to the fourth branch if they fail to answer correctly 2 consecutive questions in the third branch. It is assumed that questions in the fourth branch are easier than those in the third branch and help students prepare for the questions in the third branch. Students are transferred back to the third branch after having answering correctly 3 consecutive questions from the fourth branch.

    Students get 1 point for each question answered correctly in the first branch unless they answer correctly 3 consecutive questions in which case they automatically get 20 points and are transferred to the third branch. Students get 2 point for each question answered correctly in the third branch unless they answer correctly 3 consecutive questions in which case they automatically get an extra 40 points and the assignment ends. There is no point given from correctly answering questions in the second and fourth branch. This marking scheme may not be pedagogically good but it is amusing for the author.

    The following figure displays the policies adopted for this example.

    The maximum number of questions that can be seen before the assignment is graded is set to 20. The option algorithm is selected in both the grading and the adaptive policy sections of the menu. The algorithm to implement our adaptive policy is given below.

    
score := 0;
maxscore1 := 20*branch_weights[1];
maxscore3 := 20*branch_weights[3];
maxscore := maxscore1 + maxscore3;
 
next_branch := current_branch;
if (current_branch = 1) then
   if (streak = 3 and response_branches[1+seen-3] = current_branch) then
      next_branch := 3;
   elif (streak = -2) then
      next_branch := 2;
   fi;
elif (current_branch = 2 and streak = 3) then
   next_branch := 1;
elif (current_branch = 3) then
   if (streak >= 3 and response_branches[1+seen-3] = current_branch) then
      score := maxscore;
   elif (streak = -2) then
      next_branch := 4;
   fi;
elif (streak = 3) then
   next_branch := 3;
fi;
 
if ( score = maxscore ) then
   returnGrade(score);
elif ( seen = 12 ) then
   flag := 0;
   for i from 1 to seen do
     if ( response_branches[i] = 3 ) then
       if (flag = 0) then
         score := maxscore1;
         flag := 1;
       fi;
       if ( responses[i] = 1 ) then
         score := score + branch_weights[3];
       fi;
     elif ( response_branches[i] = 1 and responses[i] = 1 ) then
       score := score + branch_weights[1];
     fi;
   end do;
   returnGrade(score);
else
   next_branch;
fi;
