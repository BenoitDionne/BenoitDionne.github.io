Fractal Geometry

The following figure is the famous Sierpinski Gasket.

It is a typical example of a fractal object. We define in the section on fractal dimensions on the next page what is a fractal object. We do not need this definition right now.

To store this figure on a computer, we need 24235 bytes (png format). Using some simple mathematical technique (namely Iterative Functions Systems (IFS) ), this figure (to any level of accuracy) can be stored using only 552 bytes (a simple Matlab code).

The traditional way to save a (black and white) figure is to store on the computer the information for each pixel of the figure. In the next section, we explain how to create (draw) the Sierpinski Gasket and how to stored it.

Basically, the IFS technique used to create the Sierpinski Gasket can be used to create other images. We illustrate in the next sections how IFS can be used to generate different images.



Sierpinski Gasket

Iterative Function System

The Sierpinski Gasket can be generated using a simple iterative procedure that we now describe.

  • First, chose any set of points in a square: a line, a triangle, ... are fine.
  • Second, shrink the original square to half its size.
  • Third, draw back into the original square three copies of the shrunk square as illustrated in the following figures.

We repeat the second and third steps above as many times as we want (until we are happy with the figure we get). The next figure is the second iteration of the second and third steps of the IFS described above.

Independently of the set of points that you choose at the beginning, you always get the Sierpinski triangle.

Instead of storing on the computer all the pixels of the figure of the Sierpinski triangle, we only have to store on the computer (using mathematical formulae) the second and third steps of the IFS described above.

The Sierpinski triangle on the unit square is generated by the following three functions. They shrink the unit square to a square where the sides have length 1/2, and map the shrunk square back into the unit square as described above.

  • First function:
      X = x/2
      Y = y/2 + 1/2
  • Second function:
      X = x/2 + 1/2
      Y = y/2
  • Third function:
      X = x/2
      Y = y/2

There is a mathematical theorem that explains why the IFS for the Sierpinski triangle always generate the same image independently of the initial set that has been chosen. In fact, each IFS generates only one image. This theorem is called the Fixed Point Theorem . This theorem can easily be understood in the context of the simple IFS defined by the logistic mapping that can be found in the following pages .

Probabilistic System

There is a simple technique to illustrate the Sierpinski gasket. Let A the point with coordinate (0,1), B the point with coordinate (1,0) and C the origin (see the figure below). Suppose that we have an unbiased dice and that 1,6 are associated to A, 2,5 are associated to B and 3,4 are associated to C.

  • Choose any point x0 in the plane.
  • Throw the dice. Suppose that you get a 3.
  • The point x1 is the middle of the line between x0 and C.
  • Throw the dice. Suppose that you get a 1.
  • The point x2 is the middle of the line between x1 and A.
  • And so on.

The isolated points are invisible. We only see the cluster of points. This cluster of points shadows the Sierpinski gasket. To get a good approximation of the Sierpinski gasket, the number of iterations must be very large and the random number generator of your computer must be really ... really good (is it?).
